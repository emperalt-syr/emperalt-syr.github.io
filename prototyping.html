Prototyping

From storyboards to high-fidelity screens

Built in Figma across three tasks — onboarding, eligibility checking, and deadline alerts.

Tools & Process

How the prototype was built

The prototype was built in Figma using Figma AI to accelerate the screen design process. The design went through two major iterations — a low-fidelity prototype that established the basic flow and content, and a high-fidelity prototype that applied consistent visual design, added missing screens, and addressed issues identified during heuristic evaluation. All screens were designed mobile-first for iPhone with a consistent color system, bottom navigation bar, and plain-language copy throughout.

🎨

Design tool

Figma — used for both lo-fi and hi-fi prototyping, with connected flows allowing click-through navigation across all three tasks.

📐

Design system

Consistent green color palette (dark forest green for primary actions, light green for positive results, amber for deadline urgency), Inter typeface, and rounded card components throughout.

🔄

Iteration process

Lo-fi prototype → heuristic evaluation → identified issues → revised hi-fi prototype. Key changes documented in the Evaluation page.
